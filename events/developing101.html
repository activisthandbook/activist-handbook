Be lazy, use tools

  • Don't know something? Someone has already written down the answer. You just need to find it. Use Google, Stackoverflow and W3Schools
  • Editor: Just like when writing a paper in Word, there are apps for writing code. You can use apps like Visual Studio Code or Atom.
  • Package managers: Developers are lazy. We don't like writing code. So we use code that others have written a lot. Package managers are like app stores but for code. You download a piece of code that you need using the package manager.
  • Libraries: When building many apps, you will find there are parts of code you want to have available every time. Libraries are basically bundles of code that you can import in your own app, so that you don't have to write them yourself over and over.
  • Frameworks: Frameworks are similar to libraries. But instead of importing a small piece of code, frameworks are meant to be used as a blueprint for your app. We will be using a Javascript framework called VueJS.

The command promt/terminal 

Also called: shell, console

The command prompt (Windows) or terminal (Mac) is the scary thing you see in movies. You use commands to let your computer do basically anything (really anything, please don't accidentally delete all your folders). 

Frequently used commands:

  • Type dir MyFolder (Windows) or cd MyFolder (Mac) to navigate folders. To go up one folder type cd .. (two dots).
  • Type mkdir MyNewFolder followed to create a folder

Developing your first web app

  1. Create a new Vue project using the Vue Client.
  2. Open the graphical user interface by running vue ui in your command prompt/terminal. The GUI itself is also a web app, so it will open in your browser!
  3. Use the graphical user interface to serve your new project, or use the command prompt/terminal by running vue serve in your project folder.
  4. Open the folder with your code editor (e.g. Visual Studio Code).
